12 * format string validation. The logging backend can use either C++20 standard
13 * library formatting (std::format/std::println) or fmtlib (fmt::format/fmt::println)

17/**
18 * @defgroup logging Logging Module
19 * @brief Simple, type-safe logging with compile-time format string validation
20 *
21 * The logging module provides a header-only logging utility that wraps format
22 * string validation and output to stdout. It supports two backends:
23 *
24 * - **Standard library** (default when LOGENIUM_LOGGING_USE_FMTLIB=OFF):
25 * Uses std::format_string, std::println from C++20 <format> and <print>
26 *
27 * - **fmtlib** (when LOGENIUM_LOGGING_USE_FMTLIB=ON):
28 * Uses fmt::format_string, fmt::println from fmtlib v12.1.0
29 *
30 * ## Features
31 *
32 * - **Compile-time validation**: Format strings are checked at compile time
33 * - **Type safety**: Arguments are type-checked against the format string
34 * - **Source location tracking**: Automatically captures and displays file, line, and function
35 * - **Level-based logging**: Supports 10 severity levels from Trace3 to Fatal
36 * - **Perfect forwarding**: Arguments are forwarded without unnecessary copies
37 * - **Zero overhead**: Template-based design with no runtime cost
38 *
39 * ## Usage
40 *
41 * Basic logging with automatic type deduction and level selection:
42 * @code
43 * #include <logging/logging.hxx>
44 *
45 * // Default Info level
46 * logging::log("Hello, {}!", "world");
47 * // Output: [Info] Hello, world! (file.cxx:42 in main)
48 *
49 * // Level-specific logging
50 * logging::debug("Debug value: {}", value);
51 * logging::warn("Warning: {}", condition);
52 * logging::error("Error occurred: {}", error_msg);
53 * logging::fatal("Fatal error: {}", critical_error);
54 *
55 * // All levels: trace3, trace2, trace1, trace, debug, info, warn, error, fatal
56 * @endcode
57 *
58 * ## Output Format
59 *
60 * All log messages follow this format:
61 * @code
62 * [Level] message (file:line in function)
63 * @endcode
64 *
65 * ## Implementation Notes
66 *
67 * The logging module is header-only and uses template metaprogramming to provide
68 * compile-time format string validation. The actual formatting backend is selected
69 * at compile time based on the __LOGENIUM_LOGGING_USE_FMTLIB__ preprocessor macro.
70 *
71 * Source location is automatically captured via std::source_location::current()
72 * and displayed in every log message, providing precise debugging information.
73 */

106/**
107 * @ingroup logging_internal
108 * @brief Implementation struct for compile-time validated logging
109 *
110 * This struct provides the actual logging implementation with compile-time
111 * format string validation. It uses CTAD (Class Template Argument Deduction)
112 * to automatically deduce template arguments from constructor parameters.
113 *
114 * @tparam L The logging level for this log message
115 * @tparam Args Variadic template parameter pack for format arguments
116 *
117 * @note This is an implementation detail. Users should use the logging::log
118 * family of alias templates instead of instantiating LogImpl directly.
119 */
120template <Level L, typename... Args>
121struct LogImpl {
122#if __LOGENIUM_LOGGING_USE_FMTLIB__
123
124 /**
125 * @brief Construct and execute a log operation using fmtlib
126 *
127 * Validates the format string at compile time and outputs the formatted
128 * message to stdout using fmt::println with the format:
129 * `[Level] message (file:line in function)`
130 *
131 * @param format Format string with compile-time validation (fmt::format_string)
132 * @param args Variadic arguments to be formatted (forwarded)
133 * @param location Source location for debugging (automatically captured)
134 *
135 * @note Arguments are perfectly forwarded to avoid unnecessary copies
136 * @note Source location is automatically captured and displayed in output
137 */
138 LogImpl(fmt::format_string<Args...> format, Args &&...args,
139 std::source_location location = std::source_location::current()) {
140 fmt::println("[{}] {} ({}:{} in {})", L, fmt::format(format, std::forward<Args>(args)...), location.file_name(),
141 location.line(), location.function_name());
142 }
143
144#else
145
146 /**
147 * @brief Construct and execute a log operation using standard library
148 *
149 * Validates the format string at compile time and outputs the formatted
150 * message to stdout using std::println with the format:
151 * `[Level] message (file:line in function)`
152 *
153 * @param format Format string with compile-time validation (std::format_string)
154 * @param args Variadic arguments to be formatted (forwarded)
155 * @param location Source location for debugging (automatically captured)
156 *
157 * @note Arguments are perfectly forwarded to avoid unnecessary copies
158 * @note Source location is automatically captured and displayed in output
159 */
160 LogImpl(std::format_string<Args...> format, Args &&...args,
161 std::source_location location = std::source_location::current()) {
162 std::println("[{}] {} ({}:{} in {})", L, std::format(format, std::forward<Args>(args)...), location.file_name(),
163 location.line(), location.function_name());
164 }
165
166#endif
167};
