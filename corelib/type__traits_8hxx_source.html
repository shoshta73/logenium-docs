16/**
17 * @defgroup type_traits Type Traits Module
18 * @brief Custom type traits extending std::type_traits
19 *
20 * The type traits module provides compile-time type manipulation utilities that
21 * extend the C++ standard library's type traits. These are primarily used internally
22 * by the casting module but are exposed for general use.
23 *
24 * ## Components
25 *
26 * ### Const Qualification
27 * - `add_const_past_pointer<T>`: Add const past pointer indirection
28 * - `const_pointer_or_const_ref<T>`: Conditional const pointer/reference selection
29 *
30 * ### Reference Handling
31 * - `add_lvalue_reference_if_not_pointer<T>`: Conditional lvalue reference addition
32 *
33 * ### Type Classification
34 * - `is_integral_or_enum<T>`: Check if type is integral or enumeration
35 *
36 * ## Usage
37 *
38 * These traits follow the standard library conventions with `_t` suffixes for
39 * type aliases and `_v` suffixes for value traits:
40 *
41 * @code
42 * // Type transformation
43 * using T1 = add_const_past_pointer_t<int*>; // int* const (const pointer)
44 * using T2 = const_pointer_or_const_ref_t<int*>; // const int*
45 *
46 * // Type checking
47 * constexpr bool b = is_integral_or_enum_v<MyEnum>; // true
48 * @endcode
49 *
50 * Include the convenience header:
51 * @code
52 * #include <corelib/type_traits.hxx>
53 * @endcode
54 *
55 * Or include specific headers:
56 * @code
57 * #include <corelib/type_traits/add_const_past_pointer.hxx>
58 * #include <corelib/type_traits/is_integral_or_enum.hxx>
59 * @endcode
60 */
