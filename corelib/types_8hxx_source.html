15/**
16 * @defgroup types Types Module
17 * @brief Platform-independent type aliases for integer and floating-point types
18 *
19 * The types module provides consistent, portable type aliases that map to standard
20 * C++ integer and floating-point types. These aliases use a concise naming convention
21 * common in systems programming and game development.
22 *
23 * ## Integer Types
24 *
25 * Fixed-width integer types with consistent size across platforms:
26 * - **Unsigned**: `u8`, `u16`, `u32`, `u64` (8, 16, 32, 64-bit unsigned integers)
27 * - **Signed**: `i8`, `i16`, `i32`, `i64` (8, 16, 32, 64-bit signed integers)
28 * - **Signed aliases**: `s8`, `s16`, `s32`, `s64` (alternative names for signed integers)
29 *
30 * All integer types are defined using the fixed-width types from `<cstdint>`:
31 * - `u*` types use `std::uint*_t`
32 * - `i*` and `s*` types use `std::int*_t`
33 *
34 * ## Floating-Point Types
35 *
36 * IEEE 754 floating-point types with explicit precision:
37 * - **f32**: 32-bit single-precision floating-point (maps to `float`)
38 * - **f64**: 64-bit double-precision floating-point (maps to `double`)
39 *
40 * ## Usage
41 *
42 * Include the convenience header to get all type aliases:
43 * @code
44 * #include <corelib/types.hxx>
45 *
46 * u32 unsigned_value = 42;
47 * i64 signed_value = -100;
48 * f32 float_value = 3.14f;
49 * f64 double_value = 2.71828;
50 * @endcode
51 *
52 * Or include specific headers:
53 * @code
54 * #include <corelib/types/int.hxx> // Integer types only
55 * #include <corelib/types/float.hxx> // Floating-point types only
56 * @endcode
57 *
58 * ## Rationale
59 *
60 * These type aliases provide:
61 * - **Clarity**: Explicit bit-width in the type name
62 * - **Portability**: Consistent behavior across platforms
63 * - **Brevity**: Shorter names than standard library equivalents
64 * - **Consistency**: Unified naming convention across the codebase
65 */
