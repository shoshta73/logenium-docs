16/**
17 * @defgroup casting Casting Module
18 * @brief RTTI-free casting utilities inspired by LLVM
19 *
20 * The casting module provides a complete infrastructure for runtime type queries
21 * and type conversions without using C++ RTTI. This is achieved through the
22 * `classof()` pattern where types implement static type-checking methods.
23 *
24 * ## Features
25 *
26 * - **Zero overhead**: No vtables or RTTI required
27 * - **Type safe**: Compile-time type checking with runtime validation
28 * - **Flexible**: Supports pointers, references, std::unique_ptr, std::optional
29 * - **Const correct**: Preserves const-qualification throughout casts
30 * - **SFINAE friendly**: Works with template metaprogramming
31 *
32 * ## Core Operations
33 *
34 * ### Type Checking
35 * - `isa<T>(val)`: Check if value is of type T (supports variadic)
36 * - `isa_and_present<T>(val)`: Null-safe type checking
37 *
38 * ### Checked Casts
39 * - `cast<T>(val)`: Cast with assertion on failure (debug builds)
40 * - `cast_if_present<T>(val)`: Null-safe checked cast
41 * - `cast_or_null<T>(val)`: Returns nullptr on failure
42 *
43 * ### Dynamic Casts
44 * - `dyn_cast<T>(val)`: Returns nullptr on failure
45 * - `dyn_cast_if_present<T>(val)`: Null-safe dynamic cast
46 * - `dyn_cast_or_null<T>(val)`: Explicit nullable variant
47 * - `unique_dyn_cast<T>(ptr)`: For std::unique_ptr ownership transfer
48 *
49 * ### Predicate Functors
50 * - `IsaPred<T>`: Type checking predicate for algorithms
51 * - `CastTo<T>`, `DynCastTo<T>`: Casting functors for transformations
52 *
53 * ## Requirements
54 *
55 * Types must implement `static bool classof(const Base *)` for RTTI-free
56 * type checking:
57 *
58 * @code
59 * class Base {
60 * public:
61 * enum Kind { K_Base, K_Derived };
62 * Kind getKind() const { return kind; }
63 * protected:
64 * Base(Kind k) : kind(k) {}
65 * private:
66 * Kind kind;
67 * };
68 *
69 * class Derived : public Base {
70 * public:
71 * Derived() : Base(K_Derived) {}
72 * static bool classof(const Base *b) {
73 * return b->getKind() == K_Derived;
74 * }
75 * };
76 * @endcode
77 *
78 * ## Usage
79 *
80 * @code
81 * Base *b = new Derived();
82 *
83 * // Type checking
84 * if (isa<Derived>(b)) {
85 * // b is a Derived
86 * }
87 *
88 * // Checked cast (asserts in debug)
89 * auto &d = cast<Derived>(b);
90 *
91 * // Dynamic cast (returns nullptr on failure)
92 * if (auto *d = dyn_cast<Derived>(b)) {
93 * // Use d
94 * }
95 *
96 * // Variadic type checking
97 * if (isa<Derived1, Derived2>(b)) {
98 * // b is either Derived1 or Derived2
99 * }
100 * @endcode
101 *
102 * Include the convenience header:
103 * @code
104 * #include <corelib/casting.hxx>
105 * @endcode
106 *
107 * Or include specific headers:
108 * @code
109 * #include <corelib/casting/isa.hxx>
110 * #include <corelib/casting/cast.hxx>
111 * #include <corelib/casting/dyn_cast.hxx>
112 * #include <corelib/casting/predicates.hxx>
113 * @endcode
114 */

116/**
117 * @defgroup casting_detail Internal Implementation
118 * @ingroup casting
119 * @brief Internal implementation details of the casting module
120 *
121 * This group contains the internal implementation machinery that powers the
122 * public casting APIs. These components use advanced template metaprogramming
123 * techniques including SFINAE, CRTP, type traits, and recursive type
124 * simplification.
125 *
126 * **Note**: This group documents internal implementation details. Most users
127 * should use the public API (isa, cast, dyn_cast, etc.) instead of directly
128 * using these internal components.
129 *
130 * ## Architecture
131 *
132 * The implementation is organized into several layers:
133 *
134 * ### Type System (`detail/casting/traits.hxx`)
135 * - `SimplifyType<T>`: Recursive type unwrapping (removes const, references, pointers)
136 * - `IsNullable<T>`: Detects nullable types (pointers, unique_ptr, optional)
137 * - `ValueIsPresent<T>`: Presence checking for nullable types
138 * - `isPresent()`, `unwrapValue()`: Helper functions for value handling
139 * - `SelfType<T>`: CRTP helper for type customization
140 *
141 * ### Forwarding (`detail/casting/forwarding.hxx`)
142 * - `NullableValueCastFailed<To>`: Default failure handler
143 * - `DefaultDoCastIfPossible<To, From>`: CRTP base for conditional casting
144 * - `ForwardToPointerCast<To, From>`: Reference-to-pointer forwarding
145 * - `ConstStrippingForwardingCast<To, From>`: Const-aware forwarding
146 *
147 * ### Type Checking (`detail/casting/isa.hxx`)
148 * - `IsaImpl<To, From>`: Core ISA implementation using classof()
149 * - `IsaImplConst<To, From>`: Const and pointer handling
150 * - `IsaImplWrap<To, From>`: Type simplification wrapper
151 *
152 * ### Type-Specific Adapters (`detail/casting/adapters.hxx`)
153 * - `UniquePtrCast<To, From>`: unique_ptr casting with ownership transfer
154 * - `OptionalValueCast<To, From>`: Optional-returning casts
155 * - `ValueFromPointerCast<To, From>`: Pointer-to-value casting
156 *
157 * ### Cast Infrastructure (`detail/casting/cast.hxx`)
158 * - `CastRetty<To, From>`: Return type calculation
159 * - `CastIsPossible<To, From>`: Cast possibility checking
160 * - `CastConvertVal<To, From>`: Cast conversion
161 * - `CastInfo<To, From>`: Main cast orchestrator
162 *
163 * ### Predicate Functors (`detail/casting/predicates.hxx`)
164 * - `IsaCheckPredicate<T>`: ISA predicate implementation
165 * - `IsaAndPresentCheckPredicate<T>`: Null-safe ISA predicate
166 * - `StaticCastFunc<T>`, `CastFunc<T>`, `DynCastFunc<T>`: Cast functors
167 *
168 * ## Design Patterns
169 *
170 * - **CRTP (Curiously Recurring Template Pattern)**: Used for customization points
171 * - **SFINAE (Substitution Failure Is Not An Error)**: For conditional compilation
172 * - **Type Traits**: Compile-time type introspection
173 * - **Recursive Templates**: For type simplification
174 * - **Template Specialization**: For type-specific behavior
175 */
