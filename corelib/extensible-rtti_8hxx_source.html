17/**
18 * @defgroup extensible-rtti Extensible RTTI Module
19 * @brief RTTI-free runtime type identification and casting system
20 *
21 * The extensible RTTI module provides a lightweight alternative to C++ RTTI
22 * that integrates seamlessly with corelib's LLVM-style casting infrastructure.
23 * It uses compile-time type identifiers (addresses of static variables) to
24 * enable runtime type checking and safe dynamic casting without vtable overhead.
25 *
26 * Key features:
27 * - **No C++ RTTI required**: Works with `-fno-rtti` compiler flag
28 * - **Zero runtime overhead**: Type IDs are compile-time constants
29 * - **LLVM-style casting**: Full integration with isa<>, cast<>, dyn_cast<>
30 * - **Multiple inheritance**: Support for complex type hierarchies
31 * - **CRTP-based**: Automatic method generation via Extends template
32 *
33 * ## Architecture
34 *
35 * The system consists of two main components:
36 *
37 * ### Base Class
38 * Abstract base class providing the RTTI interface:
39 * - `TypeID()`: Static type identifier
40 * - `DynamicTypeID()`: Runtime type identifier
41 * - `IsA()`: Type checking method
42 *
43 * ### Extends Template
44 * CRTP template that automatically implements the RTTI interface:
45 * - Injects type-specific TypeID implementation
46 * - Overrides DynamicTypeID() with derived type
47 * - Provides IsA<T>() template for convenient type checking
48 * - Implements classof() for LLVM-style casting integration
49 *
50 * ## Usage
51 *
52 * ### Basic Hierarchy
53 *
54 * @code
55 * #include <corelib/extensible-rtti.hxx>
56 *
57 * // Base of your hierarchy
58 * class MyBase : public corelib::rtti::Extends<MyBase, corelib::rtti::Base> {
59 * public:
60 * MyBase() {}
61 * private:
62 * static char ID;
63 * void anchor() override {}
64 * friend class corelib::rtti::Extends<MyBase, corelib::rtti::Base>;
65 * };
66 *
67 * // Derived class
68 * class MyDerived : public corelib::rtti::Extends<MyDerived, MyBase> {
69 * public:
70 * MyDerived() {}
71 * private:
72 * static char ID;
73 * void anchor() override {}
74 * friend class corelib::rtti::Extends<MyDerived, MyBase>;
75 * };
76 *
77 * // Implementation (.cxx file)
78 * char MyBase::ID = 0;
79 * char MyDerived::ID = 0;
80 * @endcode
81 *
82 * ### Type Checking
83 *
84 * @code
85 * MyDerived obj;
86 * corelib::rtti::Base *base = &obj;
87 *
88 * // Using IsA() method
89 * if (base->IsA(MyDerived::TypeID())) {
90 * // base points to MyDerived
91 * }
92 *
93 * // Using template IsA<>()
94 * if (obj.IsA<MyBase>()) {
95 * // obj is derived from MyBase
96 * }
97 * @endcode
98 *
99 * ### LLVM-Style Casting
100 *
101 * @code
102 * using namespace corelib;
103 *
104 * MyBase *base = new MyDerived();
105 *
106 * // Type checking
107 * if (isa<MyDerived>(base)) {
108 * // base points to MyDerived
109 * }
110 *
111 * // Checked cast (asserts on failure in debug builds)
112 * MyDerived *derived = cast<MyDerived>(base);
113 *
114 * // Safe cast (returns nullptr on failure)
115 * if (MyDerived *derived = dyn_cast<MyDerived>(base)) {
116 * // Successfully cast to MyDerived
117 * }
118 * @endcode
119 *
120 * ### Multiple Inheritance
121 *
122 * @code
123 * class InterfaceA : public corelib::rtti::Extends<InterfaceA, corelib::rtti::Base> {
124 * // ... interface methods ...
125 * private:
126 * static char ID;
127 * void anchor() override {}
128 * friend class corelib::rtti::Extends<InterfaceA, corelib::rtti::Base>;
129 * };
130 *
131 * class InterfaceB : public corelib::rtti::Extends<InterfaceB, corelib::rtti::Base> {
132 * // ... interface methods ...
133 * private:
134 * static char ID;
135 * void anchor() override {}
136 * friend class corelib::rtti::Extends<InterfaceB, corelib::rtti::Base>;
137 * };
138 *
139 * // Multiple inheritance: inherits from both InterfaceA and InterfaceB
140 * class Implementation : public corelib::rtti::Extends<Implementation, InterfaceA, InterfaceB> {
141 * public:
142 * Implementation() {}
143 * private:
144 * static char ID;
145 * void anchor() override {}
146 * friend class corelib::rtti::Extends<Implementation, InterfaceA, InterfaceB>;
147 * };
148 *
149 * char InterfaceA::ID = 0;
150 * char InterfaceB::ID = 0;
151 * char Implementation::ID = 0;
152 * @endcode
153 *
154 * ## Design Patterns
155 *
156 * ### Type ID Pattern
157 * Each type has a unique static `char ID` member. The address of this variable
158 * serves as a compile-time constant identifier. Since each type has its own
159 * ID variable, the addresses are guaranteed to be unique.
160 *
161 * ### Anchor Idiom
162 * The pure virtual `anchor()` method ensures the vtable is emitted in a single
163 * translation unit, preventing vtable duplication and reducing binary size.
164 *
165 * ### CRTP (Curiously Recurring Template Pattern)
166 * The Extends template uses CRTP to inject type-specific implementations at
167 * compile time, eliminating the need for manual boilerplate in each derived class.
168 *
169 * ## Integration with Casting
170 *
171 * The extensible RTTI system provides the `classof()` method required by
172 * corelib's LLVM-style casting infrastructure. This enables seamless use
173 * of `isa<>`, `cast<>`, `dyn_cast<>`, and casting predicates with RTTI
174 * hierarchies.
175 *
176 * @see corelib::rtti::Base
177 * @see corelib::rtti::Extends
178 * @see @ref casting
179 */
