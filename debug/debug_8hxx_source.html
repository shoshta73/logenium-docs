15/**
16 * @defgroup debug Debug Module
17 * @brief Debug utilities for development and diagnostics
18 *
19 * The debug module provides essential debugging tools that help during development:
20 * - **Breakpoints**: Trigger debugger breakpoints for inspection
21 * - **Assertions**: Runtime checks with detailed diagnostics and stack traces
22 * - **Debugger detection**: Check if running under a debugger
23 *
24 * All debug utilities are designed to have minimal or zero overhead in release builds:
25 * - Breakpoint() becomes a consteval no-op in release mode (NDEBUG defined)
26 * - Assert() checks are completely removed in release builds
27 * - IsDebuggerPresent() remains available in all builds for runtime behavior changes
28 *
29 * ## Components
30 *
31 * ### Breakpoints
32 * - `Breakpoint()`: Unconditionally trigger a debugger breakpoint (debug builds only)
33 * - `BreakpointIfDebugging()`: Conditionally break only if debugger is attached
34 *
35 * ### Assertions
36 * - `Assert()`: Runtime checks with formatted messages and stack traces
37 *
38 * ### Debugger Detection
39 * - `IsDebuggerPresent()`: Check if the process is running under a debugger
40 *
41 * ## Usage
42 *
43 * Include the convenience header to get all debug utilities:
44 * @code
45 * #include <debug.hxx>
46 * @endcode
47 *
48 * Or include specific headers:
49 * @code
50 * #include <debug/breakpoint.hxx>
51 * #include <debug/assert.hxx>
52 * #include <debug/is_debugger_present.hxx>
53 * @endcode
54 */
