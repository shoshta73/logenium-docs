11/**
12 * @ingroup debug
13 * @brief Triggers a debugger breakpoint only if a debugger is currently attached.
14 *
15 * This is a convenience function that combines `IsDebuggerPresent()` and `Breakpoint()`
16 * to provide safe, conditional breakpoints. It checks if a debugger is attached and only
17 * triggers a breakpoint if one is present, avoiding unwanted behavior when running without
18 * a debugger.
19 *
20 * In debug builds, this function:
21 * 1. Calls `IsDebuggerPresent()` to check for an attached debugger
22 * 2. If a debugger is present, calls `Breakpoint()` to halt execution
23 * 3. If no debugger is attached, returns immediately without side effects
24 *
25 * This is particularly useful for:
26 * - **Assertion-like breakpoints**: Break on unexpected conditions during debugging, but
27 * allow normal execution in production or when running tests
28 * - **Safe breakpoints in automated tests**: Place breakpoints that won't crash test runs
29 * - **Conditional debugging**: Break at specific code points only when actively debugging
30 * - **Development convenience**: Leave debugging breakpoints in code that won't affect
31 * non-debugging workflows
32 *
33 * @note In release builds (when NDEBUG is defined), this function becomes a consteval
34 * no-op that compiles away completely, resulting in zero runtime overhead.
35 *
36 * @note Unlike `Breakpoint()`, this function is safe to call in any context (tests,
37 * production debug builds, etc.) because it only breaks when a debugger is actually
38 * attached.
39 *
40 * @note On platforms where `IsDebuggerPresent()` always returns `false` (currently Linux/POSIX),
41 * this function will never trigger a breakpoint, effectively becoming a no-op.
42 *
43 * Example usage:
44 * @code
45 * void process_transaction(const Transaction &txn) {
46 * if (!txn.is_valid()) {
47 * // Break here during development to inspect invalid transactions,
48 * // but allow tests and production to continue
49 * debug::BreakpointIfDebugging();
50 *
51 * // Handle error gracefully
52 * log_error("Invalid transaction");
53 * return;
54 * }
55 * // ... process transaction ...
56 * }
57 * @endcode
58 *
59 * @code
60 * // Example: Debug a specific iteration without affecting test runs
61 * for (size_t i = 0; i < items.size(); ++i) {
62 * if (i == 42) {
63 * // When debugging, break at iteration 42 to inspect state.
64 * // When running tests, this does nothing.
65 * debug::BreakpointIfDebugging();
66 * }
67 * process(items[i]);
68 * }
69 * @endcode
70 *
71 * @code
72 * // Example: Safe assertion-like breakpoint for development
73 * class Cache {
74 * public:
75 * void insert(const Key &key, const Value &value) {
76 * if (map.contains(key)) {
77 * // This shouldn't happen, but if it does during debugging,
78 * // break so we can investigate. In tests/production, continue.
79 * debug::BreakpointIfDebugging();
80 * }
81 * map[key] = value;
82 * }
83 * private:
84 * std::unordered_map<Key, Value> map;
85 * };
86 * @endcode
87 *
88 * @see Breakpoint() for unconditional breakpoints
89 * @see IsDebuggerPresent() to check debugger state without breaking
90 */
91void BreakpointIfDebugging();
