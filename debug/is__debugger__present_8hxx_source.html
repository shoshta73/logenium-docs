11/**
12 * @ingroup debug
13 * @brief Checks whether the process is currently running under a debugger.
14 *
15 * This function detects if a debugger is attached to the current process at the
16 * time of invocation. It can be used to conditionally enable debugging features,
17 * modify behavior for better debugging experience, or trigger breakpoints only
18 * when actively debugging.
19 *
20 * The implementation uses platform-specific mechanisms:
21 * - **Windows**: Calls the Windows API `IsDebuggerPresent()` function, which checks
22 * the PEB (Process Environment Block) `BeingDebugged` flag
23 * - **Linux/POSIX**: Currently returns `false` (stub implementation)
24 *
25 * @return `true` if a debugger is currently attached to the process, `false` otherwise.
26 *
27 * @note In release builds (when NDEBUG is defined), this function becomes a
28 * consteval function that always returns `false`, allowing the compiler to
29 * optimize away debugging-specific code paths completely.
30 *
31 * @note The result reflects the debugger state at the moment of the call. A debugger
32 * can attach or detach at any time, so the result may change between calls.
33 *
34 * @note On Windows, this function is reliable and accurate. On Linux/POSIX platforms,
35 * the current implementation is a stub and always returns `false`. Full support
36 * for Linux debugger detection may be added in future versions.
37 *
38 * Example usage:
39 * @code
40 * void log_message(const char *msg) {
41 * if (debug::IsDebuggerPresent()) {
42 * // When debugging, output detailed diagnostic information
43 * std::println(stderr, "[DEBUG] {}: {}", __func__, msg);
44 * } else {
45 * // In normal execution, use standard logging
46 * std::println("{}", msg);
47 * }
48 * }
49 * @endcode
50 *
51 * @code
52 * // Example: Conditional breakpoint based on debugger presence
53 * void process_critical_data(const Data &data) {
54 * if (!validate(data)) {
55 * // Only break if we're actually debugging
56 * if (debug::IsDebuggerPresent()) {
57 * debug::Breakpoint();
58 * } else {
59 * // Log error and continue in production
60 * log_error("Invalid data detected");
61 * }
62 * }
63 * }
64 * @endcode
65 *
66 * @code
67 * // Example: Enable verbose tracing only when debugging
68 * class Component {
69 * bool verbose_trace = debug::IsDebuggerPresent();
70 * public:
71 * void process() {
72 * if (verbose_trace) {
73 * std::println("Processing step 1...");
74 * }
75 * // ... processing logic ...
76 * }
77 * };
78 * @endcode
79 *
80 * @see BreakpointIfDebugging() which combines this check with Breakpoint()
81 */
82bool IsDebuggerPresent();

88 * @brief Consteval version of IsDebuggerPresent() for release builds.
89 *
90 * In release builds (when NDEBUG is defined), this function is a consteval function
91 * that always returns `false`. This allows the compiler to completely optimize away
92 * any debugging-specific code paths at compile time, resulting in zero runtime overhead.
93 *
94 * @return Always returns `false` in release builds.
95 *
96 * @note This is a consteval function, meaning the return value is a compile-time
97 * constant. Conditional code based on this function will be completely eliminated
98 * by the compiler in release builds.
99 */
100consteval bool IsDebuggerPresent() { return false; }
