11/**
12 * @ingroup debug
13 * @brief Triggers a debugger breakpoint unconditionally (debug builds only).
14 *
15 * When called in debug builds, this function halts execution and transfers control
16 * to an attached debugger at the point of invocation. This allows developers to
17 * inspect program state, variables, and call stacks interactively.
18 *
19 * The implementation uses platform-specific mechanisms to trigger the breakpoint:
20 * - **Clang/GCC**: `__builtin_debugtrap()` (preferred, generates optimal code)
21 * - **MSVC**: `__debugbreak()` intrinsic
22 * - **Windows (fallback)**: `DebugBreak()` Windows API function
23 * - **POSIX (fallback)**: `std::raise(SIGTRAP)` signal
24 *
25 * @note In release builds (when NDEBUG is defined), this function becomes a
26 * consteval no-op that compiles away completely, resulting in zero runtime
27 * overhead.
28 *
29 * @note If no debugger is attached when this function is called, the behavior
30 * depends on the platform:
31 * - **Windows**: The system displays a "Just-In-Time Debugging" dialog
32 * - **POSIX**: The process receives SIGTRAP and may terminate if not handled
33 *
34 * @note For conditional breakpoints that only trigger when a debugger is attached,
35 * use `BreakpointIfDebugging()` instead.
36 *
37 * Example usage:
38 * @code
39 * void investigate_bug(int value) {
40 * if (value < 0) {
41 * // Always break here in debug builds to inspect why value is negative
42 * debug::Breakpoint();
43 * }
44 * // ... rest of function ...
45 * }
46 * @endcode
47 *
48 * @code
49 * // Example: Break at a specific iteration during debugging
50 * for (int i = 0; i < 1000; ++i) {
51 * if (i == 42) {
52 * debug::Breakpoint(); // Inspect state at iteration 42
53 * }
54 * process(i);
55 * }
56 * @endcode
57 *
58 * @see BreakpointIfDebugging() for conditional breakpoints
59 * @see IsDebuggerPresent() to check if a debugger is attached before breaking
60 */
61void Breakpoint();

71 * resulting in zero runtime overhead and no code generation.
72 *
73 * @note This is a consteval function, meaning it's evaluated at compile time and
74 * produces no runtime code whatsoever.
75 */
76consteval void Breakpoint() {}
